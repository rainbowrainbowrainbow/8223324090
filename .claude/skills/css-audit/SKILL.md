# CSS Audit — Парк Закревського Періоду

Проведи повний аудит CSS-архітектури проекту (11 файлів у `css/`).

## Архітектура CSS (11 файлів)
- `base.css` — Design System v4.0, токени, змінні
- `auth.css` — логін/реєстрація
- `layout.css` — хедер, навігація, основний layout
- `timeline.css` — таймлайн бронювань
- `panel.css` — бокова панель бронювання
- `modals.css` — модальні вікна
- `controls.css` — кнопки, інпути, контроли
- `features.css` — дешборд, статистика, фічі
- `dark-mode.css` — темна тема
- `responsive.css` — медіа-запити
- `pages.css` — стилі standalone сторінок

## Що перевіряти

### 1. Невикористані селектори
- Знайди CSS-класи в `css/*.css` що НЕ використовуються в HTML/JS файлах
- Порівняй кожен `.class-name` з grep по `index.html`, `*.html`, `js/*.js`
- Виведи список зайвих класів з файлом і рядком

### 2. Дублікати
- Знайди дубльовані CSS-властивості в одному селекторі
- Знайди однакові селектори в різних файлах (конфлікти)
- Перевір inline `<style>` в HTML-сторінках на дублі з css/ файлами

### 3. Консистентність дизайн-токенів
- Знайди хардкоджені кольори (hex/rgb) що мають бути `var(--token)`
- Перевір що всі `border-radius` використовують `var(--radius-*)`
- Перевір що `font-size` використовує `var(--font-*)`
- Перевір що `box-shadow` використовує `var(--shadow-*)`
- Знайди magic numbers в spacing (не кратні 4px)

### 4. Dark mode повнота
- Порівняй світлі стилі з `dark-mode.css` — що не покрите?
- Знайди `background: #fff` або `color: #000` без dark-mode аналога
- Перевір inline стилі в HTML на dark-mode підтримку

### 5. Responsive покриття
- Знайди елементи з `position: fixed` без мобільних стилів
- Перевір `min-height: 44px` для всіх інтерактивних елементів (WCAG)
- Перевір `font-size: 16px` мінімум для `input` (iOS zoom prevention)

### 6. Розмір та продуктивність
- Порахуй кількість рядків кожного CSS-файлу
- Знайди надто специфічні селектори (>3 рівні вкладеності)
- Знайди `!important` — кожен потребує обґрунтування
